# ICU ICD-10 Coder

A single-page web tool for adult critical care. Paste a patient's history and
presentation, and it generates a reviewable, checkable list of ICD-10-CM codes
drawn from a curated ICU code set. Runs entirely in the browser — **no patient
text ever leaves the machine and there is no backend** — so it is safe to host on
a hospital's internal network.

Live site: `https://kna85.github.io/ICU-ICD10-Coder/`

## Features

- **Coder tab** — paste free-text history/presentation → generate codes, grouped
  into *Suggested* (auto-selected), *Possible – review*, and *Mentioned but
  negated* (e.g. "no GI bleed" is caught and held back). Each row shows the
  matched terms and a confidence bar. Select what you want, then **Copy selected**
  or **Download CSV**.
- **Reference list tab** — searchable, category-filterable table of every code.
- Offline keyword + clinical-synonym matching with negation handling. Understands
  common ICU shorthand (AKI, CRRT, DKA, NSTEMI, AF, CLABSI, DVT, rhabdo, etc.).
- ~300 curated ICU ICD-10-CM codes across nephrology, sepsis/ID, respiratory,
  cardiology, VTE, shock, neuro/trauma, toxicology, GI, heme-onc, endocrine,
  obstetric-ICU, and device/status codes.

## How it works

Everything lives in `index.html`. There are no dependencies, no build step, and
no network calls.

- `SEED` — the code list. Rows are `["CODE","Description","Specialty · Group"]`.
- `SYN` — clinician shorthand → canonical term mapping (add abbreviations here).
- The matcher builds a weighted keyword index from each description and scores the
  pasted text against it, applying synonym expansion and clause-level negation.

To add or edit codes, edit the `SEED` array; to teach it new shorthand, add to
`SYN`. Save and reload — no rebuild needed.

## Deploy on GitHub Pages

1. Create a new repository named `ICU-ICD10-Coder` on your GitHub account.
2. Push these files (see below).
3. In the repo: **Settings → Pages → Build and deployment → Source: Deploy from a
   branch**, branch `main`, folder `/ (root)`, then **Save**.
4. The site publishes at `https://kna85.github.io/ICU-ICD10-Coder/` within a minute
   or two.

## Hosting inside a hospital network

Because it is one static file with no backend and no external calls, it can also
be served from any internal web server (IIS/Apache/nginx/SharePoint) by placing
`index.html` in a served folder, or opened directly from a shared drive. Ask IT
for a folder on an existing intranet site, reachable on the internal network only.

## Disclaimer / accuracy

Suggestions are generated by keyword matching against a curated code set and are
intended to assist, not replace, clinical coding judgement. Codes were compiled
from ICD-10-CM reference knowledge; validate against your institution's current
ICD-10-CM/AM master before clinical use. Do not enter patient identifiers.
